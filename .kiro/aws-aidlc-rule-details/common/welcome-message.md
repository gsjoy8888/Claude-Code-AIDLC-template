# 👋 欢迎使用 AI-DLC（AI 驱动的开发生命周期）！👋

我将引导您完成一个自适应软件开发工作流，该工作流会智能地根据您的特定需求进行调整。

## 什么是 AI-DLC？

AI-DLC 是一个结构化但灵活的软件开发过程，可根据您的项目需求进行调整。可以将其视为拥有一位经验丰富的软件架构师，他会：

- **分析您的需求**并在需要时提出澄清问题
- **根据复杂性和风险规划最佳方法**
- **跳过不必要的步骤**以进行简单更改，同时为复杂项目提供全面覆盖
- **记录一切**，以便您拥有决策和理由的完整记录
- **通过每个阶段引导您**，并提供清晰的检查点和批准门

### 阶段细分：

**INCEPTION 阶段** - *规划和应用设计*
- **目的**：确定要构建什么以及为什么
- **活动**：理解需求、分析现有代码（如果有）、规划方法
- **输出**：清晰的需求、执行计划、关于并行开发工作单元数量的决策
- **您的角色**：回答问题、审查计划、批准方向

**CONSTRUCTION 阶段** - *详细设计、实施和测试*
- **目的**：确定如何构建它
- **活动**：详细设计（需要时）、代码生成、全面测试
- **输出**：工作代码、测试、构建说明
- **您的角色**：审查设计、批准实施计划、验证结果

**OPERATIONS 阶段** - *部署和监控（未来）*
- **目的**：如何部署和运行它
- **状态**：未来部署和监控工作流的占位符
- **当前状态**：构建和测试活动在 CONSTRUCTION 阶段处理

## 关键原则：

- ⚡ **完全自适应**：每个步骤根据您的需求独立评估
- 🎯 **高效**：简单更改仅执行基本步骤
- 📋 **全面**：复杂更改获得所有保障措施的完整处理
- 🔍 **透明**：您在工作开始之前看到并批准执行计划
- 📝 **记录**：所有决策和更改的完整审计跟踪
- 🎛️ **用户控制**：您可以请求包含或排除步骤

## 接下来会发生什么：

1. **我将分析您的工作区**以了解这是新项目还是现有项目
2. **我将收集需求**并在需要时提出澄清问题
3. **我将创建执行计划**，显示我建议运行哪些步骤以及原因
4. **您将审查和批准**计划（或请求更改）
5. **我们将执行计划**，在每个主要步骤设置检查点
6. **您将获得工作代码**以及完整的文档和测试

AI-DLC 过程适应：
- 📋 您的意图清晰度和复杂性
- 🔍 现有代码库状态
- 🎯 更改的范围和影响
- ⚡ 风险和质量要求

让我们开始吧！
