# 内容验证规则

## 强制要求：文件创建前的内容验证

**关键要求**：所有生成的内容必须在写入文件前进行验证，以防止解析错误。

## ASCII 图表标准

**关键要求**：在创建任何包含 ASCII 图表的文件之前：

1. **加载** `common/ascii-diagram-standards.md`
2. **验证**每个图表：
   - 计算每行字符数（所有行必须具有相同宽度）
   - 仅使用：`+` `-` `|` `^` `v` `<` `>` 和空格
   - 不使用 Unicode 制表符
   - 仅使用空格（不使用制表符）
3. **测试**对齐，验证方框角落垂直对齐

**参见 `common/ascii-diagram-standards.md` 了解模式和验证清单。**

## Mermaid 图表验证

### 必需的验证步骤
1. **语法检查**：在文件创建前验证 Mermaid 语法
2. **字符转义**：确保特殊字符正确转义
3. **后备内容**：如果 Mermaid 验证失败，提供文本替代方案

### Mermaid 验证规则
```markdown
## 在创建任何包含 Mermaid 图表的文件之前：

1. 检查节点 ID 中的无效字符（仅使用字母数字 + 下划线）
2. 转义标签中的特殊字符：" → \" 和 ' → \'
3. 验证流程图语法：节点连接必须有效
4. 使用简单验证测试图表解析

## 后备方案：如果 Mermaid 验证失败，使用基于文本的工作流表示
```

### 实现模式
```markdown
## 工作流可视化

### Mermaid 图表（如果语法有效）
```mermaid
[已验证的图表内容]
```

```

## 通用内容验证

### 创建前验证清单
- [ ] 验证嵌入的代码块（Mermaid、JSON、YAML）
- [ ] 检查特殊字符转义
- [ ] 验证 markdown 语法正确性
- [ ] 测试内容解析兼容性
- [ ] 为复杂元素包含后备内容

### 错误预防规则
1. **在使用工具/命令写入文件前始终验证**：永远不要写入未验证的内容
2. **转义特殊字符**：特别是在图表和代码块中
3. **提供替代方案**：包含可视内容的文本版本
4. **测试语法**：验证复杂的内容结构

## 验证失败处理

### 当验证失败时
1. **记录错误**：记录验证失败的内容
2. **使用后备内容**：切换到基于文本的替代方案
3. **继续工作流**：不要因内容验证失败而阻塞
4. **通知用户**：说明由于解析限制使用了简化内容
